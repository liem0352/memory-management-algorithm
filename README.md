# 可变式分区存储管理系统

C 语言实现操作系统中可变式分区分配的存储管理方案，包含三种经典内存分配算法：首次适应、最佳适应、最坏适应。项目解决了 Windows 控制台中文显示乱码问题，提供完整的 UTF-8 中文版本。

## 三种算法

| 算法 | 策略 | 特点 |
|------|------|------|
| First Fit 首次适应 | 从第一个空闲区开始查找，找到即分配 | 速度快，碎片中等 |
| Best Fit 最佳适应 | 查找满足要求的最小空闲区 | 利用率高，小碎片多 |
| Worst Fit 最坏适应 | 查找最大的空闲区分配 | 保留大块，利用率较低 |

三种算法时间复杂度均为 O(n)。

## 核心特性

- 完整中文支持（UTF-8，解决控制台乱码）
- 三种经典算法 + 动态切换
- 自动分区分割与相邻空闲区合并
- 实时显示空闲区表和已分配表
- 简洁菜单操作界面

## 数据结构

```c
typedef struct {
    int start_addr;  // 起始地址
    int size;         // 分区大小
    int status;       // 0-空闲, 1-已分配
    int job_id;       // 作业 ID，空闲时为 -1
} MemoryBlock;
```

## 编译与运行

**环境**：Windows + GCC（MinGW 推荐）+ UTF-8 控制台

```bash
# 编译
gcc -Wall -g -std=c99 -o memory_management_utf8.exe memory_management_utf8.c

# 方法一：直接运行
.\memory_management_utf8.exe

# 方法二：使用测试脚本（推荐，自动设置 UTF-8 编码）
.\test_utf8.bat
```

## 功能说明

1. **选择分配算法** — 三种算法间切换
2. **分配内存** — 输入作业 ID 和所需大小
3. **回收内存** — 释放作业，自动合并相邻空闲区
4. **显示内存状态** — 查看空闲区表和已分配表
5. **初始化内存** — 重置内存状态（总大小 1024KB）

## 使用示例

```
1. 初始化内存（1024KB）
2. 选择算法：1（首次适应）
3. 分配：作业1 200KB、作业2 300KB、作业3 150KB
4. 查看内存状态
5. 回收作业2 → 相邻空闲区自动合并
6. 再次查看内存状态
```

## 实验结果

- **首次适应**：分配快，碎片中等，利用率中等
- **最佳适应**：分配慢，小碎片多，利用率较高
- **最坏适应**：分配中等，大碎片少，利用率较低

## 技术亮点

- **中文编码解决**：`SetConsoleOutputCP(65001)` + `SetConsoleCP(65001)` + `setlocale`
- **分区分割**：空闲区大于所需时自动分割
- **分区合并**：回收时自动合并相邻空闲分区
