#可变式分区存储管理系统

##项目简介
本项目实现了操作系统中可变式分区分配的存储管理方案，包括三种经典的内存分配算法：首次适应算法、最佳适应算法和最坏适应算法。项目解决了中文显示乱码问题，提供完整的UTF-8中文版本。

##项目特色
- **完整中文支持** - 解决了控制台中文乱码问题
- **三种经典算法** - 首次适应、最佳适应、最坏适应
- **自动分区合并** - 回收内存时智能合并相邻空闲区
- **实时状态显示** - 清晰显示空闲区表和已分配表
- **用户友好界面** - 简洁直观的菜单操作界面

##功能说明

###主要功能
1. **选择分配算法** - 可以在三种算法间切换
2. **分配内存** - 为作业分配内存空间
3. **回收内存** - 释放作业占用的内存
4. **显示内存状态** - 查看当前空闲区和已分配区情况
5. **初始化内存** - 重置内存状态

###支持的算法
1. **首次适应算法（First Fit）**
  - 从第一个空闲区开始查找
  - 找到第一个满足要求的空闲区即分配
  
2. **最佳适应算法（Best Fit）**
  - 查找所有满足要求的最小空闲区
  - 尽量减少内存浪费
  
3. **最坏适应算法（Worst Fit）**
  - 查找最大的空闲区进行分配
  - 保留较大的剩余空间

##使用示例

###基本操作流程
1. 运行程序后，首先初始化内存（总大小1024KB）
2. 选择分配算法（1-首次适应，2-最佳适应，3-最坏适应）
3. 分配内存：输入作业ID和所需内存大小
4. 查看内存状态：显示空闲区表和已分配表
5. 回收内存：输入要回收的作业ID
6. 系统自动合并相邻的空闲分区

###测试用例

```
1. 初始化内存
2. 选择算法：1（首次适应）
3. 分配内存：
  - 作业1，200KB
  - 作业2，300KB
  - 作业3，150KB
4. 查看内存状态
5. 回收作业2的内存
6. 再次查看内存状态（注意分区合并）
```

##技术实现

###数据结构
```c
typedef struct {
  int start_addr;  // 起始地址
  int size;  // 分区大小
  int status;  // 状态: 0-空闲, 1-已分配
  int job_id;  // 作业ID，空闲时为-1
} MemoryBlock;
```

###核心功能
-**分区分割**：当空闲区大于所需大小时，自动分割
-**分区合并**：回收内存时自动合并相邻空闲分区
-**动态管理**：支持运行时的内存分配和回收
-**UTF-8支持**：通过Windows API解决中文显示问题

##算法复杂度分析

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|------------|------------|----------|
| 首次适应 | O(n) | O(n) | 一般应用场景 |
| 最佳适应 | O(n) | O(n) | 小作业较多的场景 |
| 最坏适应 | O(n) | O(n) | 大作业较多的场景 |

##实验结果

###性能测试
通过多次测试，不同算法的表现如下：

1. **首次适应算法**
  - 分配速度：快
  - 碎片产生：中等
  - 内存利用率：中等

2. **最佳适应算法**
  - 分配速度：慢
  - 碎片产生：多（小碎片）
  - 内存利用率：较高

3. **最坏适应算法**
  - 分配速度：中等
  - 碎片产生：少（大碎片）
  - 内存利用率：较低

###测试数据示例
```
初始内存：1024KB 空闲

分配序列：
-作业1：200KB → 首次适应算法，地址0
-作业2：300KB → 首次适应算法，地址200  
-作业3：150KB → 首次适应算法，地址500

回收作业2后：
-空闲区：200KB(地址200), 374KB(地址650)
-自动合并：574KB(地址200)
```
